<Note type="info">
  **Endianness Reminder:** All multi-byte fields (except `VarInt` and `VarLong`)
  use **big-endian**. If you’re on a little-endian system (common on x86), swap
  your bytes before sending!
</Note>

{/* ---------- Primitive Integers ---------- */}

  <TabsContent value="primitive">
    <Step>
      <StepTitle>What are Primitive Integers?</StepTitle>
      <StepContent>
        Fixed-size integers represent values with a constant number of bytes. They are the simplest building blocks for most packet fields.
      </StepContent>
    </Step>

    <Step isLast="true">
      <StepTitle>Why Endianness Matters</StepTitle>
      <StepContent>
        Big-endian means the most significant byte is sent first. On little-endian systems, use Buffer methods like `writeInt32BE` instead of `writeInt32LE`.
      </StepContent>
    </Step>

    <div className="my-4">
    | Name               | Bytes | Range                                  | Use Case Example                                     |
    | ------------------ | ----- | -------------------------------------- | ---------------------------------------------------- |
    | **Boolean**        | 1     | `false` (0x00), `true` (0x01)         | Flags like “onGround” in movement packets            |
    | **Byte**           | 1     | –128 to 127                           | Small signed values (e.g., pitch angles)             |
    | **Unsigned Byte**  | 1     | 0 to 255                              | Block IDs (legacy), particle data                    |
    | **Short**          | 2     | –32768 to 32767                       | Y-coordinate in legacy position packets              |
    | **Unsigned Short** | 2     | 0 to 65535                            | Port numbers in Handshake packet                     |
    | **Int**            | 4     | –2<sup>31</sup> to 2<sup>31</sup>–1   | Entity IDs, chunk coordinates                        |
    | **Long**           | 8     | –2<sup>63</sup> to 2<sup>63</sup>–1   | UUID segments, timestamps                            |
    </div>

  </TabsContent>

{/* ---------- Variable-Length ---------- */}

  <TabsContent value="varint">
    <p>`VarInt` and `VarLong` pack integers using 7 bits per byte, with the MSB as a continuation flag. Small values use fewer bytes.</p>

    <div className="my-4">
    | Name        | Bytes   | Encodes                  | Notes                                           |
    | ----------- | ------- | ------------------------ | ----------------------------------------------- |
    | **VarInt**  | 1–5     | 32-bit signed integer    | Read 7 bits per byte until MSB=0               |
    | **VarLong** | 1–10    | 64-bit signed integer    | Same logic, up to 10 bytes                     |
    </div>

    <Note>
      To decode, read each byte, mask with `0x7F`, shift accordingly, and repeat until the MSB is 0.
    </Note>

  </TabsContent>

{/* ---------- Strings & Text ---------- */}

  <TabsContent value="string">
    <p>All text fields are length-prefixed UTF-8. The length prefix is a `VarInt` counting UTF-16 code units (not bytes).</p>

    <div className="my-4">
    | Name                   | Prefix    | Encoding          | Limits                             |
    | ---------------------- | --------- | ----------------- | ---------------------------------- |
    | **String(n)**          | VarInt n  | UTF-8             | Up to `n` UTF-16 code units        |
    | **Identifier**         | VarInt    | UTF-8             | Resource names (letters, digits, `_`, `:`)
    | **Text Component**     | NBT Tag   | JSON/NBT          | Rich chat, hover, click events     |
    </div>

    <Note>
      JSON text must follow `{ "text": "..." }`, with optional formatting fields (e.g., `bold`, `color`).
    </Note>

  </TabsContent>
